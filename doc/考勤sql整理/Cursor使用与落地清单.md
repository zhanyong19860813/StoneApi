# Cursor 使用与落地清单（StoneApi 考勤 SQL）

## 目标

把“改存储过程 + 造测试数据 + 断言回归 + 问题定位”这条链路固定下来，减少反复试错。

---

## 一、优先安装/启用的工具

## 1) SQL Server 连接工具（高优先级）

- 目标：在 Cursor 内直接连库、查结构、跑小段 SQL。
- 你要做：
  - 安装 SQL Server 相关插件（支持连接、执行、结果查看）。
  - 配置开发库连接（只连测试库，不连生产）。
- 验收标准：
  - 能在 Cursor 里直接执行 `SELECT TOP 10 * FROM ...` 并看到结果。

## 2) SQL 格式化工具（高优先级）

- 目标：统一 `JOIN / UPDATE / CASE` 风格，减少语法和阅读错误。
- 你要做：
  - 安装 SQL formatter。
  - 约定一个统一风格（关键字大写、JOIN 对齐、每列单行等）。
- 验收标准：
  - `8_att_pro_DayResult.sql` 任意片段格式一致、可读性明显提升。

## 3) Git 变更查看工具（高优先级）

- 目标：快速确认“这次到底改了哪几处”。
- 你要做：
  - 启用 Cursor 内的 Source Control 视图（或安装 Git 可视化扩展）。
  - 每次修改后先看 diff，再执行 SQL。
- 验收标准：
  - 一次改动可在 1 分钟内定位到所有受影响文件。

## 4) CSV 预览工具（中优先级）

- 目标：快速核对 `Data_full_scenarios_对照表.csv`。
- 你要做：
  - 安装 CSV 表格预览扩展。
- 验收标准：
  - 可以直接筛选并核对某日期或某班次用例。

## 5) 浏览器日志回传工具（中优先级）

- 目标：减少“去浏览器复制 console.log 再贴给 AI”的往返成本。
- 你要做：
  - 安装 `Console Ninja`（优先）。
  - 在前端项目跑本地开发服务时观察编辑器内日志是否同步出现。
- 验收标准：
  - 前端页面触发 `console.log` 后，可在编辑器侧直接看到日志与来源文件位置。

## 6) 内置浏览器调试（中优先级）

- 目标：在 Cursor 内统一看断点、调用栈、控制台输出。
- 你要做：
  - 配置 `.vscode/launch.json`（`pwa-chrome` 的 `launch` 或 `attach`）。
  - 用 Cursor 的 Run and Debug 启动调试会话。
- 验收标准：
  - 可在 Cursor 中打断点命中，并查看 Console/Call Stack。

## 7) 自动化浏览器日志采集（中优先级）

- 目标：把浏览器错误收集自动化，避免手工复制。
- 你要做：
  - 在 Playwright 测试中监听：
    - `page.on('console', ...)`
    - `page.on('pageerror', ...)`
  - 失败时保存日志文件或 trace。
- 验收标准：
  - 跑完测试后有可复查的日志文件，AI 可直接基于日志定位问题。

## 二、最实用的 Cursor 用法（比插件更重要）

## 1) 小步改动法

- 一次只改一个规则点：
  - 先改过程逻辑
  - 再改测试数据
  - 最后改断言
- 不要一次改完所有地方再跑。

## 2) 固定执行顺序（建议贴在桌面）

1. 执行表结构变更脚本（如 `alter_*.sql`）
2. 执行存储过程脚本（如 `8_att_pro_DayResult.sql`）
3. 执行全场景脚本（`Data_full_scenarios_202603.sql`）
4. 看断言失败明细
5. 只针对失败项改预期或逻辑

## 3) 失败定位模板（每次按这个贴结果）

- 贴三段信息给 AI：
  - 失败总数（如 `失败 2 / 43`）
  - 失败明细（`act_` vs `exp_`）
  - 对应日期在“日结果汇总”的原始行

这样 AI 能在最短时间判断：
- 是逻辑错误
- 还是断言预期没同步
- 还是历史脏数据影响

## 4) 明确“双层语义”约定

- 段状态（`attStatus1/2/3`）：只表达段内真实异常
- 总状态（`attStatus`）：表达全日最终结论

把这条写进过程头部注释，后续维护不跑偏。

## 四、每次需求变更的落地 Checklist

- [ ] 需求是否明确到“段状态”和“总状态”两个层面
- [ ] 是否新增/调整了班次字段（表结构）
- [ ] 过程逻辑是否只改了必要段落
- [ ] 是否补了“触发/不触发”成对测试数据
- [ ] 是否补了断言字段与断言样例
- [ ] 是否跑通全量并确认 `全部 N 条用例符合预期`
