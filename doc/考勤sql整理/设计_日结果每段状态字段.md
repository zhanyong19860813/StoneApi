# 日结果表「每段状态」设计方案

## 一、背景

四次卡班次有两段（上午 ST1/ET1、下午 ST2/ET2），当前 `attStatus` 只反映整体，无法明确是哪一段异常。

## 二、建议新增字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| **attStatus1** | INT NULL | 第一段（上午）状态 |
| **attStatus2** | INT NULL | 第二段（下午）状态 |

### 状态取值（与 attStatus 一致）

| 值 | 含义 | 适用场景 |
|----|------|----------|
| 0 | 正常 | 该段打卡完整且无迟到早退 |
| 1 | 迟到 | 上班卡晚于计划 |
| 2 | 早退 | 下班卡早于计划 |
| 3 | 漏打卡 | 该段缺上班卡或下班卡 |
| NULL | 不适用 | 该段不存在（如两次卡班次 attStatus2 为 NULL） |

### 可选：每段迟到/早退分钟数

若需要展示具体分钟数，可再增加：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| attLate1 | INT NULL | 第一段迟到分钟数 |
| attEarly1 | INT NULL | 第一段早退分钟数 |
| attLate2 | INT NULL | 第二段迟到分钟数 |
| attEarly2 | INT NULL | 第二段早退分钟数 |

## 三、与 attStatus 的关系

- **attStatus**：保留，表示**当日整体**结果（取两段中最严重的，或沿用现有逻辑）
- **attStatus1 / attStatus2**：表示**各段**状态，用于明细展示

示例：
- 上午正常、下午缺卡 → attStatus1=0, attStatus2=3, attStatus=3
- 上午迟到、下午正常 → attStatus1=1, attStatus2=0, attStatus=1

## 四、实施范围

1. **表结构**：`att_lst_DayResult` 增加 attStatus1、attStatus2
2. **临时表**：`#t_att_lst_DayResult` 同步增加
3. **存储过程**：`att_pro_DayResult` 迟到/早退/旷工逻辑中，按段写入 attStatus1、attStatus2
4. **前端/报表**：按新字段展示每段状态
